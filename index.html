Problem & Solution

CashMate was designed to directly address the key failures of existing expense management tools. We identified four major pain points and built our objectives around solving them.

The Problem

  • ‚ùå Privacy Intrusion: Mandatory phone number registration exposes user data.
  • ‚ùå Lack of Documentation: No way to get a formal, printable PDF statement of expenses.
  • ‚ùå Rigid Payments: No support for tracking large debts paid back in installments (EMIs).
  • ‚ùå Static Interfaces: Users must manually refresh to see updates from other group members.

Our Solution (Objectives)

  • ‚úÖ Enhance User Privacy: Use email-only sign-up with secure JWT authentication.
  • ‚úÖ Provide Professional Reporting: Server-side PDF generation for auditable invoices.
  • ‚úÖ Facilitate Flexible Repayment: A unique "EMI-style" module to track partial debt payments.
  • ‚úÖ Enable Real-Time Collaboration: Integrate Socket.io so all changes are reflected instantly.

Core Features

The application is organized into five core modules. Click on any module to explore its specific functional requirements and capabilities.

User Module

  • Secure Registration: Sign up with email and password. Passwords are hashed using Bcrypt.
  • JWT-Based Login: Secure, stateless authentication with JSON Web Tokens.
  • Profile Management: Users can update their display name and avatar.

Group Module

  • Group Creation: Start a new group with a title and description (e.g., "Goa Trip").
  • Member Management: Add members to the group via email search.
  • Group Status: Ability to leave a group or remove other members (if creator).

Expense Module

  • Add Expense: Record an expense with amount, description, payer, and date.
  • Dynamic Splitting: Support for splitting expenses equally, by exact amounts, or by percentage.
  • Expense History: Edit or delete existing expenses, with updates broadcast to the group.
  • Audit Logs: All expense modifications are tracked.

Settlement Module

  • Settle Up: Record a payment between users to clear outstanding debts.
  • QR Code Generation: Instantly generate a QR code for UPI payments.
  • EMI Tracker: Unique feature to record partial payments against a large debt, recalculating the remaining balance.

Communication Module

  • Group Chat: A dedicated, real-time chat window for each group.
  • System Notifications: Automated messages in the chat ("User X added an expense of $50").
  • Real-Time Sync: Built with Socket.io for instant message delivery.

Interactive Flow: Adding an Expense

Click each step to highlight it and follow the data flow.

1. User (Frontend): User fills the "Add Expense" form and clicks Submit.
‚Üì
2. API Layer (Backend): Frontend sends a `POST /api/expense` request with data to the Express server.
‚Üì
3. Controller (Backend): The server validates the data, calculates individual shares, and prepares the database entry.
‚Üì
4. Database (MongoDB): The new expense record is saved, and user balances are updated.
‚Üì
5. Socket Layer (Backend): Server emits a `new_expense` event to all other members in that group's "room".
‚Üì
6. All Clients (Frontend): All group members receive the event, re-fetch data, and their UI updates instantly.
